# Pasos-README
Guía y ejemplo de README para proyectos en GitHub - CISCO CLASS
## ¿Qué es este repositorio?
Este repositorio contiene una guía práctica para crear y estructurar un archivo README en GitHub.

## ¿Por qué es importante?
El README es lo primero que se ve al entrar a un repositorio. Explica el objetivo, contenido y cómo usar el proyecto.

## Contenido
- Qué es un repositorio en GitHub
- Diferencia entre README de perfil y README de proyecto
- Estructura recomendada (Ciclo de Vida del Análisis de Datos)
- Ejemplo listo para copiar y adaptar

## Cómo usarlo
1. Abrí el archivo README.md (LÉAME.md)
2. Copiá la plantilla
3. Adaptala a tu proyecto y guardá cambios (Commit)
4. Editá el texto con tu información
5. Hacé clic en "Confirmar cambios..." para guardar
